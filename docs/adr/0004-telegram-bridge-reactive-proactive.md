# ADR-0004 — Интерфейс: Telegram-bridge, реактивный + проактивный

**Контекст.** Нужен мобильный интерфейс к «Второму мозгу»: задавать вопросы, кидать идеи/заметки на ходу, **и получать напоминания**. Прообраз — `pachca-codex-bridge-plan` (Pachca → bridge → Codex), но он только **реактивный** (@mention → ответ).

**Решение.** Интерфейс — **Telegram-бот «Второй мозг»** через тонкий **bridge** (FastAPI, ~150 строк) на машине пользователя, по схеме `pachca-codex-bridge-plan`:
- **Реактив:** сообщение в чат → CF Tunnel → bridge → `codex exec --resume` → ответ/запись идеи в вики → ответ в Telegram. SQLite `chat→session` для непрерывности диалога.
- **Проактив (net-new):** `scheduler` (launchd/cron) периодически запускает `codex exec`, который читает due-напоминания из `reminders/` + вики (дни рождения, подарки, встречи, идеи-к-возврату) и **сам пишет** в Telegram через Bot API.

**Почему Telegram, а не Pachca/SMS/прочее.** Личный мессенджер с мобильным клиентом, Bot API с входящими webhook'ами и исходящими сообщениями, голосовые (под будущий Whisper-ингест). Бот в выделенном чате = метафора «Второй мозг».

**Следствия.** Bridge зависит от: Telegram Bot token, `codex login`, CF Tunnel — всё **интерактивный сетап** (нельзя автоматизировать в фоне), выполняется по [setup/SETUP.md](../../setup/SETUP.md). Проактивный push качества зависит от того, включена ли машина ([ADR-0005](0005-host-v1-macbook-portable.md)).
