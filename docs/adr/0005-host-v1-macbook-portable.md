# ADR-0005 — Хостинг v1: текущий MacBook, код host-portable

**Контекст.** Прообраз (`pachca-codex-bridge-plan`) предполагал всегда-включённый Mac Mini. У пользователя на старте выделенного always-on железа нет.

**Решение.** v1 крутится на **текущем MacBook**. Весь код (`bridge`, `scheduler`, `ingest`) пишем **host-portable**: пути и токены — через `.env`, демонизация — через launchd-плисты, которые тривиально переносятся. Переезд на Mac Mini/VPS — позже, без переписывания.

**Следствия (осознанные).** Проактивные push-напоминания приходят **только когда MacBook включён и не спит** — для v1 приемлемо. Возможные митигации до переезда: `caffeinate` в рабочие часы, утренний digest при пробуждении вместо точечных пушей. SPOF и 24/7 — задача фазы «always-on host» ([CONTEXT §6 OQ-4](../../CONTEXT.md)).

**Отвергнуто на старте.** *VPS/виртуалка 24/7:* решает always-on, но подписочный `codex login` на headless-сервере капризнее (OAuth, IP-флаги за «bot backend»), и личные данные оседают на чужом сервере — против приватности.
