# ADR-0021 — Дедлайны growth-страниц (`target_date`) сканируются в дайджест напоминаний

**Контекст.** Подсистема «Поиск работы» (Фаза 1) заводит growth-страницу-роадмап с дедлайном цели во frontmatter `target_date`. Изначальный план Фазы 1 ([PRD](../../../llm-wiki-content/.scratch/jobsearch-phase1/PRD.md)) фиксировал «без нового кода планировщика» ([ADR-0007](0007-engine-spawn-and-scheduler.md)) — дедлайны предполагалось дублировать `oneoff`-блоком в `reminders/reminders.md`. Но сканер вики сегодня ходит только по person-страницам и читает `birthday:` (`birthdaysFromWiki`); `target_date` у growth-страниц нигде не сканируется. Это давало два независимых места для одной даты (frontmatter роадмапа + `oneoff`), которые надо синхронизировать вручную → рассинхрон.

**Решение.** Расширяем детерминированный скан вики: даты `target_date` у `type: growth`-страниц поднимаются в `DueItem` по образцу `birthdaysFromWiki` (новая функция, напр. `targetDatesFromWiki`, вызывается из `collectDueItems`). `target_date` становится **единственным источником** дедлайна цели роадмапа — отдельный `oneoff` для общей цели не нужен. Это **осознанное расширение** скан-кода планировщика в рамках Фазы 1, **уточняющее** [ADR-0007](0007-engine-spawn-and-scheduler.md) (который остаётся в силе по spawn/sweep/идемпотентности; меняется только «никакого нового скан-кода» — было следствие, не принцип). Скан остаётся **детерминированным** (без движка/облака), как и `birthdaysFromWiki`.

**Следствия.** В `src/scheduler/reminders.ts` появляется `targetDatesFromWiki` + ветка в `collectDueItems`; покрывается тестами (`reminders.test.ts`/`digest.test.ts`: growth `target_date` отбирается в due, попадает в дайджест, идемпотентность по дате). Промежуточные вехи (напр. дедлайн готовности резюме), у которых нет своей growth-страницы, по-прежнему живут как `oneoff` в `reminders/reminders.md` — разделение «цель ↔ под-вехи» сохраняется. `birthdaysFromWiki` и `targetDatesFromWiki` — параллельные wiki-date-источники под одним `collectDueItems`; будущие поля-даты добавляются тем же паттерном.
