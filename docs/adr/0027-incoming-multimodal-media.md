# ADR-0027 — Входящее мультимодальное медиа: распознавание изображений, файлов, голосовых

> Расширяет [ADR-0023](0023-telegram-transport-media-and-callbacks.md) (исходящий транспорт → добавляем входящий приём) и [ADR-0004](0004-telegram-bridge-reactive-proactive.md)/[ADR-0014](0014-telegram-transport-long-polling.md) (транспорт-нейтральное ядро). Уважает [ADR-0015](0015-capture-write-path-permission-posture.md) (fail-closed write-path), [ADR-0011](0011-relevance-sensitivity-filter.md) (две оси фильтра), [ADR-0007](0007-engine-spawn-and-scheduler.md)/[ADR-0009](0009-tos-safe-engine-access.md) (lethal-trifecta/ToS), [ADR-0010](0010-wiki-content-model.md) (контент-модель), [ADR-0025](0025-finance-visualization-render.md) (тяжёлая обработка — Node-side, не движок), [ADR-0008](0008-engine-claude-native.md) (engine-контракт), [ADR-0002](0002-no-embedder-pure-karpathy.md) (без ML/вектора). Ничего не `superseded`: текстовый путь и инварианты безопасности сохранены. Закрывает [CONTEXT §6 OQ-2] (голосовые/локальный Whisper). PRD: [llm-wiki-content#12].

**Контекст.** Мост понимал только текст. `extractJob` (`src/bridge/app.ts`) недавно научился брать подпись к медиа (`caption`) и помечать медиа-без-подписи `mediaOnly` → подсказка «картинки/файлы пока не читаю»; но НИ ОДИН байт содержимого не скачивается — `getFile`/download нет, `file_id` игнорируется. Движок `claude -p` — text-in: prompt идёт чистой строкой-argv; локальную картинку он не принимает (CLI 2.1.x, нет `--image`; `--file` — это `file_id` из Anthropic-сервиса, не локальный attach), аудио не транскрибирует. Постура движка ([ADR-0015](0015-capture-write-path-permission-posture.md)/[ADR-0025](0025-finance-visualization-render.md)) — `spawn-fresh`, `acceptEdits`, **без shell и без сети** → он физически не может скачать файл или вызвать внешнюю модель. Ни одного ADR про изображения/аудио/файлы/vision/OCR/STT нет — только форвард-референсы (`app.ts` «нужен ADR», [ADR-0004] «под будущий Whisper», CONTEXT OQ-2). Нужно решение, прежде чем кормить движку контент из медиа.

**Решение.** Приём трёх модальностей входящего Telegram-медиа через **редукцию-к-тексту**: медиа сводится к строке детерминированным Node-кодом моста, и дальше идёт обычным текстовым ходом — через тот же `failClosedSanitize` и тот же `Engine.run`.

- **Распознавание — локальное и Node-side, не движок.** Эталон — рендер PNG ([ADR-0025](0025-finance-visualization-render.md)): тяжёлая обработка медиа делается чистым TS-кодом моста, а не движком-через-shell. `voice`/`audio` → STT (локальный whisper.cpp, small/Metal); `document` → извлечение текста (txt/csv as-is, PDF с текстовым слоем, `docx`, `xlsx`); `photo` → OCR (локальный, напр. Tesseract). Результат каждого — **строка**, которая попадает в существующий текстовый конвейер. Движок text-in (`Engine.run(prompt: string)`, [ADR-0008]) для голоса/файлов/OCR-фото **не меняется**.
- **Инвариант маскировки сохранён ([ADR-0015](0015-capture-write-path-permission-posture.md) §2).** Любой распознанный текст (транскрипт/извлечённый/OCR) обязан пройти `failClosedSanitize` ДО движка и ДО записи в `raw/`; сбой маскера → ход отменяется целиком. Маскируем **производный текст**, не байты.
- **Только ToS-safe/локальные модели ([ADR-0009](0009-tos-safe-engine-access.md)).** STT/OCR/extract — локальные; никакого облачного STT (OpenAI Whisper API и т.п.) и никакого реюза OAuth-токена подписки в стороннем HTTP-клиенте. Единственный «облачный» шаг — обычный текст-ход в официальный `claude`.
- **Транспорт — входящая загрузка.** В `TelegramClient` добавляется входящая пара (симметрия исходящим `sendPhoto`/`sendDocument` из [ADR-0023]): `getFile(file_id) → file_path` + download по файловому URL `https://api.telegram.org/file/bot<token>/<file_path>`. Гард **20 МБ** по `file_size` ДО скачивания. Owner-only: качаем байты только для апдейтов владельца. Файловый URL содержит токен в пути — целиком в логи/ошибки не пишется.
- **`Job` несёт медиа-дескриптор.** `extractJob` поднимает `MediaRef{ kind, fileId, fileUniqueId, mimeType?, fileSize?, durationSec?, fileName? }[]` вместо булева `mediaOnly` (он становится вычислимым: `media` есть && `text` пуст). Добавляются `video_note`/`animation` (закрывают тихие drop'ы). Pre-engine слой `mediaToText` диспетчит по `kind`. Текстовый путь и приоритет `text` над `caption` не меняются. Транспорт-нейтрально ([ADR-0014]): логика в общем `extractJob`/`handleJob`, не дублируется в polling/webhook; скачивание и распознавание — только в воркере; single-flight per chat (`chatLock`).
- **Провайдеры — опциональная инъекция ([ADR-0024] паттерн).** `Transcriber`/`FileExtractor`/`ImageRecognizer` инъектируются в `BridgeState`; активны только при заданной зависимости — иначе graceful skip с текущей `mediaOnly`-подсказкой. Фичефлаги per-модальность (`MEDIA_VOICE_ENABLED`/`MEDIA_FILES_ENABLED`/`MEDIA_IMAGE_OCR_ENABLED`/`MEDIA_VISION_ENABLED`), дефолт **off**; лимиты (размер/длительность/число), бюджет/throttle (Agent-SDK-кредит, [ADR-0009]).
- **Хранение и чувствительность.** Производный текст подчиняется контент-модели ([ADR-0010]/[ADR-0017]): сырьё (транскрипт/извлечённый/OCR-текст) → иммутабельный `raw/` через `failClosedSanitize`, в `wiki/` — выжимка (verbatim не пишем). Заводится `source_class` `photo:ocr`/`voice:stt`/`file:extract`; ось A чувствительности ([ADR-0011]) применяется к производному тексту — NSFW-изображение/чужой PII → карантин (`raw/.quarantine/`), финансы/здоровье/право → `keep_redact_spans` (не карантин). В `raw/.filter-log.jsonl`/логи — только метаданные/хэш/провенанс, НИКОГДА тело. Идемпотентность по `file_unique_id` (watermark двигается только после успешной записи).
- **Изображения — гибрид.** v1: **OCR→текст** (инвариант цел — распознанный текст маскируется до облака). Фаза 2: **полноценное vision** (пиксели в модель) ТОЛЬКО через официальный `claude --input-format stream-json` (единственный ToS-safe канал картинки, [ADR-0009]), за флагом `MEDIA_VISION_ENABLED` и **отдельным ADR-аддендумом**: пиксели минуют текстовый sanitizer, поэтому аддендум обязан зафиксировать on-device пре-скрин чувствительности изображения ДО отправки, поведение в privacy-max (не отправлять), бюджет/лимиты.
- **Файлы v1 — максимум.** txt/csv/PDF(текстовый слой)/docx/xlsx + сканы-PDF и PDF без текстового слоя → постраничный OCR.

**Инварианты безопасности (сохранены).**
- **Sanitize-before-cloud ([ADR-0015](0015-capture-write-path-permission-posture.md) §2).** Транскрипт/OCR/извлечённый текст — такой же исходящий-в-облако текст, как ход; проходит `failClosedSanitize` детерминированным кодом моста ДО движка; неустранимый секрет → ход отменён. Движку маскирование не доверяем.
- **Owner-only allow-list ([ADR-0009]).** Скачивание байтов и вызов STT/OCR — только для прошедшего allow-list владельца (`chat.id`/`from.id` == owner); чужой апдейт → дроп, байты не качаем.
- **Least-privilege / минимизация blast-radius ([ADR-0007] §risks).** Движок не качает файл и не зовёт модель (cwd=приватный вики-репо, без shell/сети). Загрузка — узкий owner-only метод, без generic fetch. Парсеры (pdf/docx/xlsx) и STT/OCR-субпроцессы — с лимитами размера/времени (вредоносный файл может эксплойтить парсер).
- **Lethal trifecta ([ADR-0007]/[ADR-0017]).** Недоверенное изображение/документ — новый канал prompt-injection (текст в картинке/QR/метаданных файла), который текстовый sanitizer не видит до OCR. Митигация: распознанный текст остаётся в owner-only blast-radius, проходит маскер; приватная вика и код-сессия не сосуществуют в одном ходе.
- **Граница двух репо ([ADR-0003]).** Извлечённый из медиа текст — личные данные → только приватный `llm-wiki-content`. Тесты/примеры мультимодального пути — только синтетические/битые байты; `lint:public` exit-0 обязателен.
- **Privacy-max ([ADR-0011]).** При `engine_classification: off` пиксели/аудио не уходят в облако; путь деградирует (vision выключен; локальные OCR/STT допустимы — не покидают устройство до маскировки).

**Альтернативы (отвергнуты).**
- *Облачный STT/OCR (OpenAI Whisper API, cloud-vision).* Отвергнуто: нарушает ToS/локальность ([ADR-0009]) и «маскировка ДО облака» ([ADR-0015]) — модель увидит сырой голос/паспорт до санитайзера.
- *Слать пиксели в модель по умолчанию.* Отвергнуто для v1: vision минует текстовый sanitizer (секрет/паспорт на скрине уйдёт немаскированным). Отложено за флаг + отдельный ADR-аддендум; безопасный дефолт фото = OCR→текст→sanitize.
- *Второй санитайзер по байтам файла.* Отвергнуто (логика [ADR-0023]): бессмыслица на PNG, ложняки на CSV-финданных. Маскируем производный текст, а не байты.
- *Движок сам читает файл / дать движку путь.* Отвергнуто: least-privilege ФС ([ADR-0007]) — cwd движка = приватный вики-репо, произвольные пути он не читает; извлечение делается вне процесса движка.
- *Отдельный тип `Job` под медиа.* Отвергнуто: опциональное поле `media: MediaRef[]` переиспользует очередь/воркеры/single-flight без дублирования транспорт-нейтрального ядра.
- *Tier-2 ML-классификатор чувствительности для медиа сейчас.* Отложено ([ADR-0002]/CONTEXT OQ-6): тяжёлая ML-зависимость в лёгком ingest-ядре; v1 — Tier-1 детерминированный + `source_class`.
- *Агрегация альбома `media_group` в один ход.* Отложено: Telegram шлёт элементы альбома отдельными апдейтами, `extractJob` без состояния между ними; v1 — поэлементно.
