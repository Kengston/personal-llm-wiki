# ADR-0011 — Фильтрация контента: чувствительность (NSFW/приватное) + релевантность, две оси + лейн задач

**Контекст.** Цель системы — помощник в профессиональном и личностном **росте** плюс приёмник «лишних задач» (билеты, покупки). Аудит ([research](../research/)) подтвердил: фильтра чувствительного контента (NSFW) и релевантности **нет**, `sanitizer.py` покрывает только секреты/PII, а лейна реактивных задач (отдельно от датированных `reminders/`) тоже нет. Это пробел: в вики попадает всё подряд, включая нежелательное и шум.

**Решение.** Вводим фильтр на **двух ортогональных осях + роутер-лейн**, и НИКОГДА не сливаем их в один гейт «выкинуть плохое/ненужное» (главная ошибка такого дизайна).

- **Ось A — чувствительность (on-device, до облака).** Детерминированный классификатор `ingest/classifier.py` рядом с sanitizer на write-path (НЕ внутри него — противоположная семантика отказа: секреты fail-**closed**/abort, чувствительность fail-**to-quarantine**). Маршрутизирует **целые документы** в `raw/.quarantine/<категория>/`. v1 — **только Tier-1 детерминированный** (`source_class` + `domain_blocklist` + opt-in `lexicon`); **локальная ML-модель (Tier-2) — ОТЛОЖЕНА** в отдельный будущий ADR (ломает stdlib-only-ingest; триггер ADR-0002 — это FTS5, а не нейросеть). Без embedder.
- **Ось B — релевантность/важность.** **Не новый гейт**, а расширение существующего решения compile «промоутить-vs-оставить»: добавляем диспозиции **settle** (полежать в `raw/`, переоценить позже) и **leave-in-raw**; важность = детерминированные сигналы (recurrence, owner-authored, maps-to-entity, decision-language, survives-settle) + LLM-целое 1–10 как тай-брейкер для пограничных. Egress тут уже оплачен compile-шагом → новой утечки нет.
- **Роутер «задача vs знание» (ДО важности).** Эфемерные чоры (билеты/покупки) → `raw/.tasks/inbox/` + тонкий `tasks/log.md`, выполняются реактивно, **не судятся на важность и не роняются**; ростовой сигнал **не** выкидывается как чора. Роутер **консервативен в сторону знания**: дивертит только при форме «императив + действие-объект», на сомнении — **дуал-роут** (и в task-log, и видимо для compile). Карантин **побеждает** лейн (чувствительная чора → карантин, не в `.tasks/`).

**Категории (дефолт, owner-tunable в [`compiler/relevance-policy.md`](../../compiler/relevance-policy.md)).** Карантин: **NSFW**, **чужие персданные**, **токсик/абьюз** (никогда авто-дроп). **ХРАНИМ как знание + редактим только опасные подстроки** (по требованию владельца — это жизненно-важные данные): **финансы** (долги/кто-кому — храним смысл, sanitizer маскирует номера карт/IBAN), **здоровье** (цели/заметки), **право** (смысл). Settle/в raw: низкая важность/шум/доскролл.

**Инварианты и фиксы (вшиты).** `raw/` иммутабелен → «drop» переопределён как «не промоутить в `wiki/`», **хард-delete нет**; re-promote из карантина — один логируемый шаг (escape-hatch для false-positive). **Карантин предпочитает ложно-ПОЛОЖИТЕЛЬНЫЕ** (переусердствовать дёшево, `raw/` хранит, байты остаются локально); только *drop-from-wiki* — ложно-отрицательные. Каждое действие → строка в append-only `raw/.filter-log.jsonl` + `log.md` (verb `filter`); **в лог — только категория/score/provenance, НИКОГДА содержимое**. Карантин/`.tasks/` — поддиректории внутри иммутабельного `raw/`, **исключаются из compile/query/digest/resurface явным пропуском dot-папок** (`rglob` сам их НЕ пропускает — нужен явный `part.startswith('.')`). Карантин-ревью и дайджест читают **лог/метаданные, не тела** (изоляция инъекций). Политика разнесена: публичная схема/дефолты — `compiler/relevance-policy.md`; приватные кастом-лексиконы/имена — в приватном репо (`.filter-policy.local.json`), мёрджатся в рантайме. Движок **никогда** не редактирует политику автономно. Privacy-max режим (`engine_classification: off`) полностью отключает облачную классификацию.

**Следствия.** Новый `ingest/classifier.py`; новый `compiler/relevance-policy.md`; правки `compiler/rules.md` (новые §8b/§5a, verb `filter`, запреты), `ingest/llm_chat.py` (вызов классификатора на **write-site целого документа**, не в per-message цикле), `scheduler/digest.py` + routine `quarantine-review`, и явный dot-exclusion во всех читателях `raw/`. Наследует fail-closed-контракт sanitizer, границу двух репо ([ADR-0003](0003-two-repos-public-private.md)), no-embedder ([ADR-0002](0002-no-embedder-pure-karpathy.md)), контент-модель ([ADR-0010](0010-wiki-content-model.md)). Tier-2 ML и продвинутый task-актуатор — будущий ADR.
