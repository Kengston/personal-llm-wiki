# ADR-0013 — Фикс pii_density: считаем только валидные телефоны (даты ≠ телефон)

**Контекст.** Порт фильтра на TypeScript ([ADR-0012]) был behavior-preserving — и
вместе с поведением перенёс пред-существующий баг Python-оригинала. Ось A
чувствительности ([ADR-0011]) считает «плотность PII» (`pii_density`) как число
совпадений email- и phone-regex; на пороге (дефолт 5) категория `others_pii`
уходит в карантин. Phone-regex `(?<!\d)\+?\d[\d\s().-]{8,}\d(?!\d)` ловит **фрагмент
даты** ISO-таймстемпа — `2024-05-29` (8 цифр) распознаётся как «телефон». Каждая
ingested-страница (llm_chat / telegram) несёт МНОГО таймстемпов: frontmatter
(`date_range`, `ingested_at`) + per-message stamps в расшифровке. Поэтому почти
любая страница перебирала порог 5 и **ложно** уходила в `others_pii`
(`quarantine_and_redact`), выпадая из compile/query/digest. Это ломало основной
поток ингеста — а сердце вики именно диалоги с LLM ([ADR-0010]).

**Решение.** В `piiDensity` телефоном считаем только совпадение, прошедшее проверку
«реального номера» — **10..15 значащих цифр** (хелпер `looksLikePhone`, тот же, что
в sanitizer-ярус-2; экспортирован → один источник правила телефона). `2024-05-29` →
8 цифр → НЕ телефон. Реальные номера (RU/E.164, ≥10 цифр) считаются как прежде.

Это **осознанное расхождение** с faithfully-ported Python — фикс бага, НЕ редизайн
фильтра. Оси, диспозиции, направление ошибок, карантин-побеждает-лейн ([ADR-0011])
не меняются: устраняется лишь ложно-положительный сигнал в одном детекторе.

**Следствия.** Дат-насыщенные страницы идут нормальным путём, не в карантин.
Регрессионный тест (`src/ingest/classifier.test.ts`) фиксирует это + позитивный
контроль (5 реальных телефонов → `others_pii`). Карантин по-настоящему
чувствительного (NSFW/чужие персданные/токсик через `source_class`/`keyword`/
`lexicon`) — без изменений. **Уточняет, не отменяет** [ADR-0011]; Python-оригинал
удалён ([ADR-0012]), синхронизировать не с чем.
