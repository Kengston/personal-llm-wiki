# ADR-0006 — Публичный репо на личном GitHub-аккаунте; создание через `gh auth login`

**Контекст.** Нужно захостить публичное портфолио-репо. Машина уже авторизована в GitHub по SSH под нужным личным аккаунтом — push работает «из коробки». `gh` CLI не установлен. (Присланный изначально публичный SSH-ключ для пуша не используется: его приватной половины на машине нет, а подписывает пуш именно приватная половина — публичный ключ это то, что *добавляют на* аккаунт, а не то, что *подписывает с* машины.)

**Решение.** Публичный `personal-llm-wiki` и приватный `llm-wiki-content` хостим на одном личном GitHub-аккаунте, под которым машина уже залогинена по SSH (приватный — с флагом `--private`).

**Создание репо.** SSH умеет только push в существующий репо, не создаёт его. Поэтому: **`brew install gh` + `gh auth login`** (выбрать аккаунт в браузере) → `gh repo create` создаёт и пушит. Альтернатива без `gh`: создать пустой репо в web-UI и `git push` по существующему SSH.

**Следствие (приватность).** Личные идентификаторы — email'ы, имена аккаунтов, пути к ключам — в публичный репо **не попадают**: это операционные детали, их место в приватной заметке или `.env`, не в портфолио. Гард — `scheduler/lint_public.py` (скан PII/секретов перед коммитом) + корневой [`.gitignore`](../../.gitignore).
