# ADR-0009 — ToS-безопасный доступ к движку (официальный бинарь, single-user)

**Контекст.** Anthropic в Jan–Apr 2026 **заблокировала сторонние OAuth-харнессы** (OpenClaw, OpenCode, Goose, Roo Code…), которые переиспользуют подписочный токен. При этом **официальный Claude Code headless под подпиской явно разрешён** Consumer Terms (автоматизация запрещена «кроме как через API-ключ ИЛИ где явно разрешено» — Claude Code и есть это разрешение) и описан как «ordinary individual use». Источники — [research/privacy-security](../research/privacy-security.md).

**Решение.** Движок вызывается **только через официальный бинарь** (`claude -p`), под аккаунтом владельца. Жёсткие правила:
- **Single-user:** allow-list на собственный Telegram `chat_id`; задачи шлёт только владелец (мультиюзер = нарушение account-sharing).
- **Никогда** не скрейпить/реюзать OAuth-токен в свой/сторонний HTTP-клиент — это ровно то, что банят (паттерн OpenClaw).
- **Human-in-the-loop / умеренные расписания**, не 24/7 always-on (это не бан, но триггерит недельные лимиты).
- Приватно, не как «сервис/бот для других».

**Стоимость (важно).** С **15.06.2026** скриптовый Claude Code / Agent SDK на подписке тянет из месячного **Agent-SDK-кредита** (~$100/мес на Max-5x, per-user); сверх — по API-ставкам. Для персонального масштаба щедро, не бесконечно.

**Для отложенного Grok-адаптера.** Grok гоняется официальным **Grok Build CLI** или через **OpenClaw** — последний для Grok **xAI-санкционирован** (в отличие от Claude, где OpenClaw забанен). То есть OpenClaw допустим только на Grok-стороне, никогда на Claude.

**Следствия.** Bridge спавнит официальные бинари → safe by construction. Масштабирование/мультиюзер/коммерция → переключение на metered API (Commercial Terms, без ограничений на автоматизацию).
