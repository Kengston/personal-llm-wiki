# ADR-0002 — Без embedder и векторной базы (чистый Karpathy LLM-wiki)

**Контекст.** Соблазн на старте — прикрутить RAG/векторную базу для «семантического поиска» по личным заметкам.

**Решение.** **Никаких embedder'ов и vec-индексов ни в одной фазе.** Знание хранится как persistent markdown-вики; навигация — через `index.md` и кросс-ссылки; семантическое ранжирование делает **сам LLM-клиент** (Codex), читая вики. При росте корпуса — добавляем лексический **SQLite FTS5**, но не векторы. Наследует `abcage-wiki` ADR-0008.

**Почему.** В этом и суть паттерна Карпатого: знание **компилируется один раз** в связную вики, а не re-retrieve'ится из чанков на каждый запрос. Вектора возвращают фрагменты без накопления; вики накапливает синтез, кросс-ссылки и помеченные противоречия. Для личного корпуса (сотни страниц) `index.md` + LLM-ранжирование достаточно и радикально проще в эксплуатации.

**Следствия.** Нулевая инфраструктура поиска на старте. Trip-wire на пересмотр: если LLM-клиент стабильно не находит релевантные страницы через `index.md` на корпусе > N страниц — тогда вводим FTS5 (всё ещё не вектора).
