# ADR-0008 — Движок v1: Claude-native, engine-portable, Grok/Codex — отложенные адаптеры

**Контекст.** Серия гриллинг-итераций (см. [log](../../log.md)) уточнила факты и цели: у владельца есть **Claude Max ($100)**, ChatGPT-подписки нет. Разбор показал, что Claude в этой системе закрывает **все три роли** — интерактивный мозг (Telegram-реактив), «руки» (web/computer-агент: динамические JS-сайты, research), и **компилятор вики** (плановые правки десятков md-файлов). Claude Code headless под подпиской **ToS-нативен** (официальный бинарь, single-user — см. [ADR-0009](0009-tos-safe-engine-access.md)) и **уже оплачен**. Grok рассматривался как доверенный «голос-советчик», но это мягкое предпочтение, а не закрытый capability-gap.

**Решение.** v1 — **Claude-native, один движок**: `claude -p --output-format json` (официальный бинарь). Bridge остаётся **engine-portable**: абстракция `Engine` + дефолт **`ClaudeEngine`**; **`GrokEngine`** и **`CodexEngine`** — готовые **отложенные адаптеры** (слоты), добавляются конфигом без переписывания. Grok подключается позже как опциональный **advisor-голос** для A/B жизненных советов (его единственная уникальная ценность — субъективное доверие владельца, релевантна сердцу вики про развитие).

**Отвергнутые альтернативы.**
- *Codex primary ([ADR-0001](0001-engine-subscription-codex.md)):* нет ChatGPT-подписки.
- *Один Grok Build ($30):* Grok слабее как web/computer-агент (Anthropic первой сделала зрелый computer-use; Grok ~17 пунктов SWE-bench позади на агентном кодинге), и не переиспользует оплаченный Max.
- *Двух-мозговый роутер (Claude-doer + Grok-advisor):* избыточная сложность для v1 — Claude покрывает обе роли; роутер тривиально доращивается, когда/если добавим Grok-голос.

**Следствия.** `$0` сверх Max (с учётом Agent-SDK-кредита — [ADR-0009](0009-tos-safe-engine-access.md)). Bridge спавнит официальный `claude` (ToS-safe). Роли: реактив (Telegram) + «руки» (web/computer через MCP) + компилятор (routines). Контент-модель — [ADR-0010](0010-wiki-content-model.md).
