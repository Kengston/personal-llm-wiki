# ADR-0010 — Контент-модель вики: концепции/развитие/идеи-first, код → accomplishment

**Контекст.** Вики — это «второй мозг» **про владельца**, а не код-документация. Сердце — **концепции, личностное развитие, идеи**. Техническое должно быть **сильным, но сжатым и вторичным**: важно не *как* написан код, а *что* предметно сделано — чтобы вики моделировала «на что я способен» и «что реализовывал раньше».

**Решение.**
- **Типы страниц:** `ideas/` · `concepts/` · `growth/` (развитие, маркеры, цели) · `people/` · `projects/` (accomplishment-записи) · `capability-profile` (деривативный профиль «на что способен») · `journal/`.
- **Источники:** заметки + **чаты со ВСЕМИ LLM** (ChatGPT/Claude/Grok), выгрузка + **постоянная инкрементальная подгрузка** по [watermark](../research/data-ingestion.md); позже — мессенджеры/экспорты.
- **Правило сжатия (ключевое):** код/технические сессии **не пишем verbatim**. Извлекаем «**что сделал предметно**: построил X через Y, показывает навык Z, ключевые решения/уроки» → агрегируется в `capability-profile`. Полные технические детали остаются в `sources/` (raw), в `wiki/` идёт выжимка.

**Следствия.** `compiler/rules.md` кодирует типы страниц + правило сжатия кода. `ingest/` получает **llm_chat-коннектор** (парсинг экспортов ChatGPT/Claude/Grok → идеи/концепции/решения + accomplishment-выжимки). Баланс: техническая сила вики = в `capability-profile` и `projects/`, а не в простыне кода.
