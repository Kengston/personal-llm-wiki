# ADR-0001 — Движок: подписочный Codex CLI, не per-token API и не локальная модель

> **⚠️ SUPERSEDED ([ADR-0008](0008-engine-claude-native.md), 2026-05-31).** Гриллинг-итерация показала: (1) ChatGPT-подписки у владельца нет, а Claude Max — есть; (2) Claude покрывает и роль «рук» (web/computer-агент), и компилятора вики (routines), и интерактивного мозга, при этом ToS-нативен и уже оплачен. Движок переведён на **Claude-native** ([ADR-0008](0008-engine-claude-native.md)); Codex остаётся как **портируемый отложенный адаптер** в engine-абстракции. Текст ниже сохранён как история решения.

**Контекст.** «Второй мозг» нужен бюджетный (фиксированная стоимость), но с сильным ризонингом — ядро нагрузки это не чат, а агентное редактирование десятков markdown-файлов вики при каждом ingest.

**Решение (отменено).** Движок = **Codex CLI в headless-режиме** (`codex exec --json --resume`) под подпиской **ChatGPT Plus/Pro**. Авторизация через подписочный OAuth, не API-ключ → `$0` сверх подписки. 1:1 движок из `pachca-codex-bridge-plan`.

**Отвергнутые тогда альтернативы.** Per-token API (дорого при росте); локальная open-weight (слабее на file-editing); Claude Code (тогда ошибочно считали, что подписки Claude нет).

**Почему отменено.** Факт-чек: ChatGPT-подписки нет, Claude Max ($100) есть. Claude-native проще, нативнее, ToS-безопаснее и переиспользует уже оплаченное. См. [ADR-0008](0008-engine-claude-native.md), [ADR-0009](0009-tos-safe-engine-access.md).
