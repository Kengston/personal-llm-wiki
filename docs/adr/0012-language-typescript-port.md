# ADR-0012 — Язык реализации фреймворка: порт Python → TypeScript (идиоматичный, engine-portable сохранён)

**Контекст.** Язык реализации фреймворка **никогда не фиксировался как решение** — Python был дефолтом черновика, а не выбором ([log.md](../../log.md), ADR 0001–0011 язык не упоминают). Сложился слоистый footprint: stdlib-only ingest-ядро (`sanitizer.py`/`classifier.py`/`watermark.py`), FastAPI/uvicorn/httpx/structlog-bridge, `python-dateutil`-scheduler. У владельца профильный TS-стек ([`abcage-mcp-hub`](../../../PhpstormProjects/abcage-mcp-hub): Node 22 + TypeScript strict ESM + Fastify + better-sqlite3 + pino + zod + vitest, pnpm) — единый тулчейн снижает контекст-свитчинг и даёт зрелую экосистему (валидация, тесты, типы). Ключевой факт: **язык не несущий** — думает движок (Claude Code, [ADR-0008](0008-engine-claude-native.md)), а наш код — тонкий клей: текст-обработка (маскер/классификатор/коннекторы) + HTTP-мост + плановые скрипты. Поэтому это **behavior-preserving переписывание, не редизайн**.

**Решение.** Портируем **весь публичный фреймворк** (`bridge`/`ingest`/`scheduler`/`lint_public`) на **идиоматичный TypeScript** (Node **≥24** — нужен для встроенного `node:sqlite`; ESM, `strict`), зеркаля house-style `abcage-mcp-hub`. Делаем **in-place** в `~/llm-wiki` — `personal-llm-wiki` становится TS-фреймворком, Python остаётся в git-истории (одно каноническое репо вместо двух параллельных). Зависимости выбираем **по стандартам TS/MCP-серверов, а не по искусственному «zero-dep»**: Fastify/pino/zod/dotenv-flow — мост; **встроенный `node:sqlite`** для стора `chat→session` (идиоматичный Node 24+, без нативного аддона/`node-gyp` — на Node 25 у `better-sqlite3` нет prebuild'а; стор за интерфейсом `SessionStore`, бэкенд — свап одного файла); `rrule`/`js-yaml` — планировщик (идиоматичный аналог `python-dateutil` + парс frontmatter); встроенные `RegExp`/`String.normalize('NFKC')`/`node:crypto` — текст-ядро (идиоматичный выбор инструмента, **не** правило чистоты). Публичные контракты функций сохраняются 1:1 (`sanitizeText`/`scanSecrets`/`failClosedSanitize`/`classifySensitivity`/`routeLane`/`shouldSkipRawPath`/`filterLogRecord`/`loadPolicy`).

**Что СОХРАНЯЕТСЯ дословно (инварианты несёт ПОВЕДЕНИЕ, а не язык — [CONTEXT §3](../../CONTEXT.md)).** Движок — только официальный бинарь `claude`, single-user, без реюза OAuth-токена ([ADR-0008](0008-engine-claude-native.md)/[ADR-0009](0009-tos-safe-engine-access.md)); engine-portable шов (абстрактный `Engine` + `ClaudeEngine` + отложенные `GrokEngine`/`CodexEngine`); spawn-fresh-per-task ([ADR-0007](0007-engine-spawn-and-scheduler.md)); без embedder/вектора ([ADR-0002](0002-no-embedder-pure-karpathy.md)); граница двух репо + `lint_public` exit-0 ([ADR-0003](0003-two-repos-public-private.md)); `raw/` иммутабелен + fail-closed-порядок маскера; P0-фиксы фильтра ([ADR-0011](0011-relevance-sensitivity-filter.md): dot-containment через `shouldSkipRawPath`, карантин-побеждает-лейн, keep_redact_spans для финансов/здоровья/права, аудит-лог только метаданные); контент-модель ([ADR-0010](0010-wiki-content-model.md)); watermark-после-успешной-записи. JSON-блок [`compiler/relevance-policy.md`](../../compiler/relevance-policy.md) переиспользуется как есть (язык-агностичен).

**Отвергнутые альтернативы.**
- *Оставить Python.* Расходится с TS-стеком владельца; два тулчейна для одного человека; нет выигрыша, раз язык не несущий.
- *Искусственно сохранить «stdlib-only»-ядро в TS* (hand-roll RRULE/YAML/HTTP-сервер на `node:http`). Неидиоматично, больше кода, выше риск багов — а гарантии даёт **поведение + тесты**, не отсутствие зависимостей. Аудитируемость ядра сохраняется через узкий, покрытый тестами модуль, не через нулевой счётчик пакетов.
- *Новый репозиторий / side-by-side `ts/`.* Выбран in-place: одно каноническое `personal-llm-wiki`, история в git.

**Следствия.** **Ничего не supersedes** — язык не был зафиксированным решением (ADR 0001–0011 остаются в силе, замораживаются как исторический канон). spawn-fresh-per-task реализуется через `node:child_process` (`execFile`, аргументы массивом, **без shell** — минимизация blast radius, [ADR-0007](0007-engine-spawn-and-scheduler.md)); HMAC webhook-секрета — `node:crypto.timingSafeEqual` (констант-тайм, как `hmac.compare_digest`); SHA-256 аудит-лога — `node:crypto`. **Главный технический риск:** Unicode-семантика regex — Python `re` юникод-aware (`\b`/`\w` ловят кириллицу), JS `RegExp` `\w`/`\b` ASCII-only, а текст вики **русский** → наивный порт изменил бы маскирование. Закрывается осознанным портом каждого паттерна (явные классы / флаг `u` где нужно) + переносом self-test-векторов Python в vitest **плюс кириллические boundary-кейсы**. Живые доки (`AGENTS.md`/`CONTEXT.md`/`README.md`/`setup/SETUP.md`), launchd-плисты и shell-обёртки обновляются под Node/TS; ADR-история не переписывается. Порт катится фазами (ядро+тесты → мост → коннекторы+планировщик+линт → доки) с `code-review`/`security-review` до любого пуша.
