# ADR-0025 — Рендер финвизуалов: Node-side chartSpec → SVG → PNG

> Расширяет [ADR-0018](0018-finance-module.md) (финансовый модуль), [ADR-0023](0023-telegram-transport-media-and-callbacks.md) (транспорт медиа), [ADR-0015](0015-capture-write-path-permission-posture.md) (capture-write-path); не supersedes ни одного.

**Контекст.** Финансовый модуль ([ADR-0018]) и его реактивный диспетчер ([ADR-0024]) дают `chartSpec(kind,data) → ChartSpec` — нейтральную к рендереру сериализуемую спеку (line/bar/pie/progress: net-worth во времени, балансы, расходы по категориям, прогресс цели, структура долгов). Транспорт ([ADR-0023]) умеет доставлять PNG владельцу через `sendPhoto`. Между ними — дыра: нужно превратить `ChartSpec` в байты PNG.

Очевидный путь — «движок сам рисует график своим кодом» (matplotlib/любой рендер-скрипт) — **невозможен** при текущей постуре движка. По [ADR-0015] (capture-write-path) движок claude — spawn-fresh, с `acceptEdits`, но **без shell и без сети**: он не может запустить процесс-рендерер, не может поставить/вызвать matplotlib, не может скачать шрифты/бинари. Любой «движок-через-shell» нарушил бы permission-постуру и принцип single-user blast-radius ([ADR-0007]).

**Решение.** Рендер делается **на стороне Node**, двумя ступенями с чистой границей:

1. **`renderChartSvg(spec) → string`** (`src/ingest/finance/svg.ts`) — **ЧИСТАЯ** детерминированная функция: вся геометрия и вёрстка графика (оси, сетка, путь линии, сектора pie через arc-path, столбцы bar, трек+заливка progress, легенда, экранирование лейблов). Без сети, без файловой системы, без `Date.now()`/`Math.random()` — один и тот же `ChartSpec` даёт побайтово один и тот же SVG. Шрифты — generic `sans-serif` в `<text>` (без внешних зависимостей; растеризатор подставит системный фолбэк или деградирует грациозно). Пустые/placeholder-спеки (`meta.empty`) → валидный SVG с overlay «нет данных», без throw.

2. **`renderChartPng(spec) → Buffer`** (`src/bridge/finance-render.ts`) — **ТОНКИЙ адаптер**: `renderChartSvg(spec)` → растеризация в PNG. Никакой арифметики/геометрии здесь нет — только превращение строки SVG в байты.

**Выбор растеризатора — `@resvg/resvg-js`.** Чистый Rust/NAPI с prebuilt-бинарём: **без** системного librsvg, **без** headless-браузера, **без** сети. Бинарь проверен в целевом окружении (Node 25, darwin/arm64) — рендерит валидный PNG (магия `0x89 0x50 0x4E 0x47`). Это и есть несущая способность для проактивных и реактивных визуалов (доставка через `sendPhoto`, [ADR-0023]). Запасной вариант (если бинарь не грузится на иной платформе) — `sharp`; на момент решения не понадобился.

**Инварианты (не нарушать):**
- Вся «умная» часть (вёрстка/геометрия/детерминизм) — в чистой тестируемой `svg.ts`; растеризация — тонкий адаптер. Граница чистое/нечистое держится строго.
- Без сети, без spawn, без фоновых процессов — только локальная либа.
- Огрубление подписей (secret-gate, [ADR-0011]) делается **не здесь**: данные огрубляет вызывающий код **до** `chartSpec`; `svg.ts` рисует ровно то, что в спеке. `assertNoSecrets` применяется к видимому тексту caption/filename на доставке ([ADR-0023]), **не** к байтам PNG (это продукт владельцу).
- Мультивалютность ([ADR-0018]): нативные валюты as-is; pie с несравнимыми валютами деградирует на равные сектора-плейсхолдеры (доли = 0 уже в `chart.ts`).
- Нет PII в коде/тестах: только синтетические/fake-данные.

**Альтернативы (отвергнуты):**
- *Движок рендерит график (matplotlib / рендер-скрипт через shell).* Невозможно: движок без shell/сети ([ADR-0015]); нарушает permission-постуру.
- *Headless-браузер (puppeteer/playwright) для SVG→PNG.* Тяжёлый рантайм, скачивает Chromium (сеть), хрупкий на portable-хосте ([ADR-0005]); избыточен для статичного графика.
- *Системный librsvg (rsvg-convert).* Системная зависимость вне Node-дерева, не self-contained, ломает portable-деплой.
- *Готовая charting-либа с canvas-бэкендом (chart.js + node-canvas).* `node-canvas` тянет нативные Cairo/Pango системные зависимости; своя SVG-вёрстка чище тестируется (детерминированный стринг без бинаря) и не добавляет тяжёлую транзитивную цепочку.

**Следствия:**
- Новый файл `src/ingest/finance/svg.ts` (чистая вёрстка SVG) + тест `svg.test.ts` (валидность каждого типа, пустые данные, детерминизм, XML-безопасность).
- Новый файл `src/bridge/finance-render.ts` (адаптер растеризации) + смоук `finance-render.test.ts` (PNG-магия, реальный рендер без сети).
- Новая зависимость `@resvg/resvg-js` в `package.json`/`pnpm-lock.yaml`.
- Доставка визуала владельцу собирается из этого слоя + `sendPhoto` ([ADR-0023]); проактивные/реактивные визуалы (дайджест, ответ на финвопрос) переиспользуют `renderChartPng`.
